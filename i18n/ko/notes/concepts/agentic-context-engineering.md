# 에이전트형 컨텍스트 엔지니어링

## 요약

에이전트형 컨텍스트 엔지니어링은 맥락이 사용을 통해 좋아진다고 봐요.
에이전트는 맥락을 소비하기만 하지 않아요.
실행 피드백을 바탕으로 노트, 규칙, 검증을 고치는 제안을 도와요.

## 패턴

스스로 개선되는 맥락 loop에는 네 단계가 있어요.

1. 기존 맥락을 써서 답하거나 행동해요.
2. 비어 있거나, 낡았거나, 중복되었거나, 위험한 맥락을 발견해요.
3. 작고 정확한 업데이트를 제안해요.
4. 검증한 뒤 개선된 맥락을 커밋해요.

## 적용

AI Context as Code는 에이전트의 업데이트가 명시적이고 검토 가능해야 해요.
시스템은 비공개 원본을 조용히 복사하지 않으면서 노트와 규칙을 개선해야 해요.
