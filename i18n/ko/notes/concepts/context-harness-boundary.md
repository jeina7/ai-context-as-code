# 컨텍스트와 하네스의 경계

## 요약

맥락 층과 agent harness는 관련이 있지만 같은 것은 아니에요.
맥락 층은 오래 남길 지식과 규칙을 저장해요.
harness는 에이전트가 도구를 쓰고, 권한을 지키고, 실행 상태를 남기게 하는 실행 환경이에요.

## 경계

acac.sh는 맥락 층으로 시작해요.

아래 기능을 관리하기 시작할 때 harness에 가까워져요.

- 도구 접근
- 작업 상태
- 실행 기록
- 검증 보고서
- 권한 경계
- 사람이 중간에 개입한 기록
- 커밋 또는 pull request 작업 흐름

## 의미

맥락 원본이 신뢰할 수 있기 전에는 harness 기능을 먼저 붙이지 않아요.
약한 맥락 층은 실행 환경을 디버깅하기 더 어렵게 만들어요.
