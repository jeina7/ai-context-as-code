# AI Context as Code 설계

## 요약

AI Context as Code는 사람과 AI 에이전트가 함께 쓰는 마크다운 기반 맥락 시스템이에요.
처음에는 제품 저장소로 시작하고, 자기 자신의 개발 맥락을 첫 번째 문서 묶음으로 삼아요.
나중에는 핵심 원본 형식을 바꾸지 않고도 조직용 맥락 시스템으로 커질 수 있어요.

## 설계

시스템에는 다섯 층이 있어요.

- 작성 층: `notes/`와 `conventions/`
- 안전 점검 층: `private-staging/`, 안전 점검, 검증
- 빌드 층: 메타데이터, 링크, 백링크, 검색, 생성된 JSON
- 배포 층: 정적 GitHub Pages 사이트
- 에이전트 실행 기준 층: symlink로 모은 규칙, 명령, 검토 흐름, 나중에 추가할 스킬 또는 기억 참조

## 경계

이 저장소는 UI가 붙은 비공개 볼트처럼 보이면 안 돼요.
자기 자신의 검토된 맥락을 첫 자료로 가진 제품처럼 보여야 해요.

그래서 제품 코드, 제품 결정, 예시 맥락은 함께 있을 수 있지만 역할이 분리되어야 해요.

- 제품 코드는 시스템을 실제로 쓸 수 있음을 보여줘요.
- 제품 결정은 시스템이 왜 존재하는지 설명해요.
- 맥락 문서 묶음은 시간이 지나며 유용한 지식을 어떻게 포착하는지 보여줘요.
- 에이전트 실행 기준은 에이전트가 작동할 때 무엇을 직접 읽을 수 있는지 보여줘요.

이 문서 묶음은 비공개 볼트 전체를 복사한 것이 아니에요.
비공개 원본 없이도 이해되는, 검토되고 일반화된 자료만 담아요.
