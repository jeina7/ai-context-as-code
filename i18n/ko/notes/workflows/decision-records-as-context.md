# 결정 기록을 컨텍스트로 쓰기

## 요약

결정은 단순한 기록이 아니에요.
미래의 에이전트가 같은 질문을 근거 없이 다시 열지 않게 하는 맥락이에요.
좋은 결정 기록은 선택, 버린 대안, 당시 그 선택이 타당했던 이유를 설명해요.

## 패턴

시스템의 형태를 바꾸는 선택이 생기면 결정 노트를 써요.

노트는 아래 질문에 답해야 해요.

- 무엇이 바뀌었나요?
- 왜 이 선택인가요?
- 무엇을 의도적으로 하지 않았나요?
- 미래의 에이전트가 바꾸기 전에 무엇을 확인해야 하나요?

## 적용

이 저장소는 이름, 로컬 우선 구현, GitHub Pages, 빌드 과정이 맥락이 되는 이유 같은 프로젝트 선택을 `notes/decisions/`에 남겨요.
