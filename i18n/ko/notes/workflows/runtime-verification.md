# 실행 환경 검증

## 요약

작업 흐름은 스크립트가 존재한다고 증명되지 않아요.
중요한 환경에서 스크립트가 실행되고, 그 결과를 확인할 때 증명돼요.
AI가 관리하는 시스템에는 실제 사용과 가까운 검증이 필요해요.

## 패턴

자동화되거나 생성되는 시스템을 바꾼 뒤에는 세 층을 검증해요.

- 원본 검사가 통과해야 해요.
- 생성된 산출물을 다시 만들어야 해요.
- 사용자가 여는 경로에서 결과를 불러올 수 있어야 해요.

정적 지식 시스템에서는 노트 검증, 메타데이터 재생성, 사이트 로딩 확인이 여기에 해당해요.

## 적용

`scripts/review_context.py`는 이 저장소의 로컬 실행 검증 시작점이에요.
안전 점검, 노트 검증, 메타데이터 빌드를 한 번에 실행해요.
