# Why Context as Code

## Summary

The project name is AI Context as Code because the system treats AI context as a versioned, reviewable, testable artifact.
The name is descriptive rather than branded.
It connects the project to context engineering without overstating the current implementation as a full agent harness.

## Decision

Use `AI Context as Code` as the project name and `ai-context-as-code` as the repository name.

## Why

`Context` is more accurate than memory because the system is not only storing facts. It is shaping what an AI agent can see, reuse, and improve.

`as Code` is accurate because notes, conventions, validation scripts, and generated metadata live in a repository and pass checks before deployment.
