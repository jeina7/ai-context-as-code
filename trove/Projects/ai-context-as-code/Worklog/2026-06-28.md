The local dist server was reachable only from the Mac mini because it bound to `127.0.0.1`.
The server script now accepts a `--host` option so the same output can be served on the LAN during manual QA.
This is still a local development helper and does not change deploy behavior.

- Added `--host` to `scripts/serve_dist.py`.
- Verified `0.0.0.0:4173` listens on all interfaces.
- Verified root, search, and note routes still return the app shell through the fallback server.
- Moved the running dev server into the `tmux` session `acac-serve` so it stays alive outside the Codex tool session.
