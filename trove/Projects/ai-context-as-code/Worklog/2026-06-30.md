## 01:22 — Visual Direction Grill Decisions Captured

ACAC visual design system을 기존 reader 구현이나 이전 design system 문서 기준 없이 새로 grill했어요.
이번 기록은 구현 계획이 아니라, Visual Positioning Note로 옮길 수 있는 방향 결정을 정리한 거예요.

- ACAC의 시각 성격은 정물/보석 컨셉을 참고하되, 본질은 정교한 제작 도구이자 technical context system으로 잡았어요.
- Desktop App과 Web은 공통 token과 개념 surface를 공유하되, layout과 density는 분리하기로 했어요.
- Desktop은 account-level Home, Trove Home, Gem editor/reader, Forge workspace로 나누기로 했어요.
- Gem 읽기는 일반 폴더 기반 파일시스템 tree 중심으로 두고, Quarry, Chronicle, Relations, Ledger는 tree 밖 전용 surface로 분리하기로 했어요.
- 3D icon asset은 개념 설명과 사이드바 section header에만 쓰고, Gem row마다 반복하지 않기로 했어요.
- Refine loading은 실제 작업 단계와 crafted status phrase를 함께 쓰되, 결과는 validation, Ledger, Chronicle 중심으로 설명하기로 했어요.
- 기본 UI 문법은 panels, rows, logs 중심으로 두고, card 남발과 기존 문서 앱 clone 느낌을 피하기로 했어요.
- Markdown body는 문서형 renderer로 유지하고, ACAC의 검증, 관계, 기록, 결과 UI는 본문 밖 tool surface로 분리하기로 했어요.
- Frontmatter는 기본 노출하되, 판단에 필요한 필드는 가독성 좋게 보이고 내부 필드는 접기로 했어요.

다음 산출물:
