## 00:55 — Account Hierarchy And Project Terminology Corrected

ACAC product model에서 account hierarchy와 Trove 내부 분류 용어를 수정했어요.
이전 tree는 `Forge`, `Troves`, `Domains`, `Ledger`, `Chronicle`을 사용자 content tree에 너무 직접적으로 넣고 있었고, 새로 정한 선택 가능한 Forge/Trove 모델과 맞지 않았어요.

- Account hierarchy를 `<selected-forge>`와 `<selected-trove>` 중심으로 바꿨어요.
- `Domain` 용어를 제거하고 Trove 내부 단위를 `Project`로 바꿨어요.
- Trove content tree는 `Projects/<project>/<section>/<gem>.md`로 정리했어요.
- Quarry는 Trove별 raw input surface로 유지하되, Project/Gem tree와 분리했어요.
- Ledger는 `.acac/ledger/` 아래 raw append-only system history이자 work history source of truth로 정리했어요.
- Ledger Entry는 intent, operations, evidence, validation, actor, approval state, revert data를 담는 revert 단위로 정리했어요.
- Chronicle은 Ledger에서 파생되는 read-only app history view이며, Gem tree folder나 editable source가 아니라고 명시했어요.

검증:

- `python3 scripts/deploy_check.py` passed after the hierarchy update.
